About the SOLID Event-Driven Graphic Architecture

From PyQt5 Desktop to Modern HTML/CSS/JS Responsive Web Solution

🚀 Architectural Principles

This web application is the direct HTML/CSS/JS evolution of our custom **SOLID Event-Driven Graphic Architecture**. It decouples UI rendering from data models through a centralized pub/sub **`TimePeriodRegistry`**.

✨ Zero-Wiring Reactivity (`TimeAware`)

Whenever the user changes granularity (`Fiscal Weeks`, `Days`, `Months`) or selects a work cell in the top pinned filter bar, `TimePeriodRegistry` broadcasts a `TimeSpanContext`. Every active widget (`SqdpBoardWidget`, `BurndownChartWidget`, `ProgressBarChartWidget`) automatically subscribes and updates its geometry and chart canvas without manual event wiring!

📏 Dynamic Multi-Granularity Board

The `SqdpBoardWidget` adapts intelligently: when under `Fiscal Weeks`, it renders a 13-column quarterly grid (`Week 1..13`). When switched to `Days` or `Months`, it seamlessly morphs into a 31-column daily tracking matrix (`Day 1..31`).

⚠️ Intelligent Burndown Protection

Sprint Burndown curves require sprint-capable temporal boundaries (`Fiscal Weeks`). If a user switches to `Days` or `Months`, our burndown widget instantly prevents curve distortion by displaying a clean, explanatory safety notice while all other widgets continue rendering daily actuals.

🎨 Premium Obsidian & Executive Themes

Designed with curated HSL color tokens, glassmorphic headers, drop shadows, and micro-animations to deliver a rich, wow-factor user experience across any device or display resolution.
